  3. with 문의 사용

    1. with 문을 사용하면 SyntaxError가 발생한다. with 문은 전달된 객체를 스코프 체인에 추가한다. with 문은 동일한 객체의 프로퍼티를 반복해서 사용할 때 객체 이름을 생략할 수 있어서 코드가 간단해지는 효과가 있지만 성능과 가독성이 나빠지는 문제가 있다 따라서 with문은 사용하지 않는 것이 좋다.

strict mode 적용에 의한 변화

  1. 일반 함수의 this

    1. strict mode 에서 함수를 일반 함수로서 호출하면 this에 undefined가 바인딩 된다. 생성자 함수가 아닌 일반 함수 내부에서는 this를 사용할 필요가 없기 때문이다. 이때 에러는 발생하지 않는다.

    2. ```javascript
      (function() {
      ‘use strict’;

      function foo () {
      console.log(this); // undefind
      }
      foo();

      function Foo() {

      console.log(this); // foo

      }
      new Foo();
      }());
