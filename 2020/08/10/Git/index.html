Git 강의

  1. $ 입력받을 준비가 됐다 없다면 뭔가가 실행중인것
  2. ~ 최상위 폴더
  3. Ls -a/l a는 숨김
  4. Clear
  5. pwd 현재 쓰고 있는 디렉토리를 절대경로로 출력
  6. Echo “” 따옴표 안에 내용 출력
  7. Mkdir 디렉토리명 디렉토리(폴더) 생성
  8. Touch 파일명 엑셀같은 애들은 생성 불가
  9. Mv(move) style.css(파일명) bin(위치)
  10. cp(copy) style.css(파일명) ..(상위폴더) 복사
  11. copy 사본의 이름 , move 위치이동 하면서 이름변경(rename) mv style.css index.js
  12. rm(remove삭제) rm index.* > index 들어간거 다 삭제
    1. rm *.py py 확장자 다삭제
    2. rm은 폴더삭제는 불가능
    3. rm -r 안에 오브젝트를 다 지우고 자기자신까지 삭제 (오브젝트가 남아있으면 폴더가 지워지지 않는다 경로를 잃기 떄문에)
    4. / 경로에서는 어떤 것도 삭제하면 안된다
    5. sudo키워드를 쓸 땐 꼭 잘 확인할 것
  13. chmod 777 style.css 권한부여 0부터 8까지 8진수를 3개의 숫자로 권한
  14. Drwx(나)rwx(유저그룹)rwx(게스트) rwx 는 421 원하는걸 더한다 r,w만 키고 싶다면 4,2를 더해서 chmod 666 파일명
  15. open 파일명 기본확장자로 열어준다
  16. Cat READE.me = 안의 내용을 터미널에 텍스트로 제공받는다.
  17. Gitignore 깃에 올라갈 때 제외시키는 목록

클론해오는거

  1. 협업하는 방법

    1. collaboration >> settings > manage access 권한 부여 > 내 레포를 남에게 권한 부여 (권장하지 않는 방식)
    2. pusll request 팀원이 짠 코드를 팀장레포에 추가
      1. 팀장 레포에 있는걸 포크떠서 내 레포에서 클론 시작
      2. 내가 할 내용을 팀장 레포에 issues에 적어주기
      3. git branch 브렌치명 - 브렌치 생성
      4. git checkout 브렌치 - 해당 브렌치로 이동
      5. 작업
      6. Add,Commit,Push
      7. Merge
      8. Add,Commit,Push
      9. Git remote add pmorigin 팀장 코드를 최신화 하기 위해 팀장 레포를 추가로 연결해 준다.
        1. git pull pmorigin master 병합 시 오류가 뜨면 수정 해준후 Add,Commit,Push
    3. 팀장이 할일
      1. 레포 생성
      2. 생성된 레포 폴더생성후 클론
      3. 기본적인 문서 생성 및 작성 및 Add,Commit,Push
      4. 팀원이 pull request를 올리면 해당 내용 확인해서 팀장 파일에 merge후 팀원에게 재 배포

  2. git branch - 브렌치 리스트

    1. -a

  3. git branch 만들 브렌치 이름. –내가 있는 위치를 기준으로 생성

  4. git checkout 이동할 브렌치이름 – 브렌치이동

  5. git merge 병합할 브렌치 이름 — 브렌치를 병합 시킴, 병합된 브렌치는 존재하고 복사해서 붙여넣어진다

  6. Git branch -D(대문자) 삭제할 브렌치 이름 — 용도가 끝난 브렌치 삭제

  7. git checkout – . –현재 브렌치의 가장 최근 커밋으로 되돌아가기

  8. Mv templates/headr.html templates/partials >> 파일 디렉토리 이동 (삭제후 재생성)

  9. git mv templates/headr.html templates/partials >> 파일 디렉토리이동(이름 변경 방식 이동, 권장방식)
