복습

변수에 선언과 초기화, undefined가 할당되는 시점

  1. 어제 공부한내용은 변수의 선언을 함과 동시에 초기화가 진행되면서 undefined값이 할당된다고 나와있지만 사실은 변수를 선언하고 그 할당된 값을 참조하는 순간에 undefined가 할당된다 , 처음부터 이렇게 설명하면 초보자가 접근하기에 어려울 수 있기 때문에 많은 책에 이부분이 빠져있다.

키워드 ( keyword)

  1. 자바스크립트 엔진의 명령어를 의미한다
  2. 변수나 식별자는 실행컨텍스트에서 등록된다
    1. 실행컨텍스트란 자바스크립트 엔진이 소스코드를 평가하고 실행하는 환경과 그 결과를 관리하는 영역이다
    2. 실행컨텍스트를 통해서 자바스크립트 엔진은 식별자와 스코프를 관리한다 .

var,let,const 초기화를 진행하는 이유와 키워드의 선언,초기화 순서

  1. 이전 애플리케이션의 메모리 값이 존재할 수 있기 때문에 초기화를 진행해준다 .
  2. var키워드의 경우 런타임이전에 선언과 동시에 초기화를 진행하고 값에 접근하는 순간 undefined가 할당된다
  3. Let의 경우 런타임 이전에 선언단계가 시작하고 이후에 초기화가 진행되면서 스코프의 시작지점부 초기화 단계 시작지점 까지 일시적 사각지대가 생겨난다
    2. 위와 같은 구조이기 때문에 var와 다르게 런타임 이전에 변수를 참조할 경우에 ReferenceError가 뜨면서 호이스팅 현상이 발생하지 않는 것처럼 보인다
  4. const의 경우 변경할수 없는 값인 상수로 변수의 선언과 동시에 값의 할당이 이루어지기 때문에 초기화단계가 필요없다.

값을 재할당 할 시 메모리의 변화

  1. 변수 선언 시 초기화를 진행하면서 존재하던 undefined에 새로운 값을 넣어주었을 때
  2. undefined가 있던 메모리셀이 초기화되면서 새로운 값이 들어가지 않고
  3. 새로운 메모리 셀에 새로운 값이 추가되고 변수는 새로운 값이 추가된 메모리셀의 공간의 주소를 기억한다
  4. 그리고 이전에 존재했던 undefined는 추적하는 변수가 존재하지 않기 때문에 가비지 콜렉터에 의해 메모리가 해제된다
    1. 단 해제되는 시점은 알 수 없다 그렇기 때문에 초기화가 진행되야하고 초기화를 하지 않을 경우 해제되지 않은 메모리에 접근할 수 있게된다

가비지 콜렉터 (Garbage collector)

  1. 가비지 콜렉터란 사용되지 않는 메모리를 주기적으로 검사하여 해제 시켜주는 기능이다.

언매니지드 언어와 매니지드 언어

  1. C같은 언어는 메모리를 직접적으로 관리 할 수 있기 때문에 언매니지드 언어로써 가비지 콜렉터가 존재하지 않는다.
    1. 개발자의 역량에 따라 최고의 효율을 낼 수 있지만 반대의 경우 치명적인 오류가 생길 수 있다.
  2. 우리가 사용하는 javascript언어는 메모리를 직접적으로 관리 할 수 없기 때문에 매니지드 언어로써 가비지 콜렉터가 존재한다.
    1. 성능면에서 언매니지드 언어보다 불편하지만 메모리를 직접 관리하지 않는다는 점에서 편리하다.

표현식 (Experession)

  1. 표현식은 값으로 평가될 수 있는 문(statement)이다, 표현식이 평가되면 새로운 값을 생성하거나 기존값을 참조한다 .
  2. 리터럴은 값으로 평가, 리터럴 = 표현식
  3. 값으로 평가되는 모든 문 = 표현식

문

  1. 프로그램을 구성하는 가장 기본단위이며 최소 실행단위
  2. 문법적으로 더이상 쪼개질게 없는 코드의 기본 요소

자바스크립트

  1. 자바스크립트는 본래 html,css를 동적으로 움직이게 하는 보조적인 역할로서 개발자의 사용을 고려하지 않고 만들었기 때문에 사용하기 쉽게 만들기 위해서 간단하게 만들다 보니 다른언어들과는 태생적인 차이가 있다
    1. 다른 언어는 프로그래머가 시킨 일만 동작 하지만 자바스크립트 엔진은 스스로 판단하여 동작하는 일이 있다
      1. 예를 들어 1+’2’를 하게 될 경우 다른 언어는 스트링과 넘버로 구분하여 타입이 맞지않아 오류를 발생시키지만 자바스크립트 엔진은 사용자가 실수한 걸로 판단해서 숫자열을 문자열로 바꿔 ‘12’로 출력시킨다 .
      2. 이렇듯 자바스크립트 엔진은 타 언어에 비해 느슨하게 만들어 졌기 때문에 엔진이 암묵적으로 하는 행동들을 알아야한다
      3. 그런 문제를 인지하지 못한다면 남이 작성한 코드의 의미를 파악할 수 없고 그로인해 잘못된 코드를 작성하여 버그가 발생하는 상황이 생길 수 있다.
      4. 변수의 호이스팅현상이 의도적으로 만들어진 이유는 개발자가 아닌 사용자가 자바스크립트 언어를 쓸 때 오류가나서 프로그램이 멈추는것을 고려해서 변수 선언을 호이스팅시켜서 멈추지 않도록 의도했다 .
      5. 호이스팅 현상이 일어나는 것은 변수의 선언문을 포함한 모든 선언문이고
      6. 이후에 추가된 let과 const역시 호이스팅이 일어나지만 일어나지 않는 것 처럼 작동한다 .
        1. 그 이유는 let의 경우 변수의 선언과 초기화의 단계가 나눠져있기 때문이고
        2. const의 경우는 변수를 선언함과 동시에 값이 할당 되어야 하기 때문에 초기화 단계를 거치지 않아서 이다 .

SPA(single page application)

  1. CBD (component based development) 방법론을 기반으로하는 spa가 대중화되면서 angular, react, vue.js ,svelte 등 다양한 spa 프레임워크 라이브러리 또한 많은 사용층을 확보하고 있다

  2. single page application이란 단일페이지로 기존의 변경사항이 있을 시 모든 페이지를 새로고침 하는 방식에서 필요한 데이터만 서버로 부터 받게하여 트래픽의 총량을 줄이고, 새로고침 시 화면이 깜빡거리는 등의 개선된 사용자의 경험을 제공

Node.js

  1. 기존에 브라우저에만 존재했던 자바스크립트 엔진을 분리해서 서버사이드 애플리케이션 개발에 사용할 수 있게 됐다
  2. alert같은 코드는 브라우저의 경고창을 출력해주는 코드이기 때문에 브라우저가 존재하지 않는 node.js에서는 동작하지 않는다
  3. 이처럼 node.js에서만 동작하고 브라우저환경에서만 동작하는 javascript code가 존재한다
  4. ECMA script는 node.js와 브라우저환경에서 둘다 작동하는 코드만 가지고 있다

Javascript에서 파일을 생성하거나 지우는 code

  1. 브라우저 환경에서의 자바스크립트에는 파일을 생성하거나 지우는 기능이 없다, 만약 있다면 개발자는 사용자의 컴퓨터에 접근을 할 수 있게 되기 때문에 브라우저 환경에서는 파일을 생성하거나 지우는 코드가 존재하지 않는다.
  2. node.js 환경에서는 브라우저가 존재하지 않기 때문에 파일을 생성하거나 지우는 코드가 존재한다
  3. 자바스크립트 엔진은 인터프리터 언어이기 때문에 컴파일을 하지 않아도 됐었지만 v8엔진 이후 속도 개선을 하면서 일부 많이사용되는 코드를 컴파일 한다 .
