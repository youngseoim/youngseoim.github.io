프로젝트 진행 시 문제점 및 개선 사항

  1. jpg 파일이 너무 남용되서 svg 파일로 변환해서 반응형 작업 시 이미지가 깨지지 않도록 하고 상대적으로 용량이 적은 이유 때문에 선택하려고 했으나 고해상도의 이미지라 svg로 변환이 되지 않음

  2. 헤더의 로고 이미지 배치 시 서브메뉴 배경에 준 psition:absolute가 헤더의 fixed를 기준으로 배치가 되서 100%를 차지해야 하는데 로고이미지에 absolute를 주게 될 경우 로고 이미지를 기준으로 배치가 되면서 100%를 차지 할 수 없는 현상이 생겨서 float으로 변경

  3. 메인 영역

    1. 시즌음료

      1. 기존에 모든 텍스트가 이미지로 처리되어 있어서 성능 향상을 위해 텍스트로 입력하였으나 텍스트의 인라인 특성 때문에 반응형 작업 시 레이아웃이 깨지는 문제가 생겨서 stay chill stay playful 영역만 텍스트로 준후 이미지로 대체 했다
      2. 메인 시즌음료 영역에 음료이미지 3개와 음료설명 이미지 3개를 transform에 translator로 위치를 잡아주려고 했으나 6개의 이미지에 하나하나 좌표값을 줘야하고, 또 translator는 자기자신이 원래 있던 위치에 영역을 차지하고 이미지만 옮겨지는 것이기 때문에 문제가 생길 수 있다고 판단 하여 훨씬 공수가 적은 position을 사용하고 이미지와 설명을 하나로 랩핑한 후 배치했다

    2. 공지사항 / 프로모션

      1. 공지사항에 리스트를 javascript를 사용하지 않고 css animation 효과로만 배치하려고 했으나 리스트를 끝내고 첫번째 리스트로 다시 돌아 올 때 딜레이가 생김, 추후에 javascript를 배우고 기능구현 할 예정

    3. 리워드

      1. 성능 향상을 위해 이미지를 텍스트로 대체 하려고 했으나 기존에 백그라운드 이미지를 배치하는 과정에서 텍스트가 있으면 서로 밀어내면서 배치가 되지 않는 현상이 발생해서 이미지로 대체 했다.

  4. 다른 팀들과 다르게 반응형을 목표로 하다보니 조급함이 느껴져서 처음엔 영역을 나눠서 진행을 했는데 생각해보니까 이번 프로젝트의 목표는 프로젝트를 끝내는 것 보단 공부를 하는게 주 목적이라는 생각이 들어서 헤더영역부터 하나하나 서로 토론해가고 공부해가면서 만들 다 보니 좀 더 나은 방향으로 개선도 되고 커뮤니케이션의 중요성을 느낄 수 있었다.
