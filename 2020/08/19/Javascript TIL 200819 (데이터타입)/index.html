데이터 타입

  1. 원시 타입 (primitive type)

    1. 숫자 (number): 정수 실수의 구분 없이 하나의 숫자 타입만 존재

      1. c나 자바의 경우 정수와 실수를 구분해서 다양한 숫자타입을 제공한다
      2. 자바스크립트는 배정밀도 64비트 부동소수점 형식을 따르며 정수만 표현하기 위한 데이터 타입은 존재하지 않으며 모든 수를 실수로 처리한다.
      3. 추가적으로 세 가지 특별한 값도 표현 가능하다
        1. infinity : 양의 무한대
        2. -infinity : 음의 무한대
        3. NaN : 산술 연산 불가 (자바스크립트는 대소문자를 구별하기 때문에 대소문자를 구별해서 적어야한다)

    2. 문자열 (string): 문자열

      1. 작은따옴표,큰따옴표 또는 백틱 으로 텍스트를 감싸고 가장 일반적인 표기법은 작은따옴표를 사용하는 것이다

      2. 문자열을 따옴표로 감싸는 이유는 키워드나 식별자 같은 토근과 구분하기 위해서 이다.

      3. 자바스크립트의 문자열은 원시타입이며 변경 불가능한 값이다

      4. 템플릿 리터럴(template literal) : ES6 부터 도입된 새로운 문자열이다

        1. 멀티라인 문자열 (multi-line string)

          1. 일반 문자열 내에서는 줄바꿈(개행)이 허용되지 않는다 따라서 일반 문자열에서 줄바꿈 등의 공백을 표현하려면 백슬래시로 시작하는 이스케이프 시퀸스(escape sequence)를 사용해야 한다.

          3. 예를 들어, 줄바꿈과 들여쓰기가 적용된 HTML 문자열은 다음과 같이 이스케이프 시퀀스를 사용해 작성한다.

            일반 문자열과 달리 템플릿 리터럴 내에서는 이스케이프 시퀀스를 사용하지 않고도 줄바꿈이 허용되며, 모든 공백도 있는 그대로 적용된다.

        2. 표현식 삽입 (expression interpolation)

          1. 문자열은 문자열 연산자 +를 이용해 연결할 수 있다 +연사자는 피연산자 중 하나 이상이 문자열인 경우 연결 연산자로 동작하고 그외는 덧셈 연산자로 동작한다.

          2. 템플릿 리터럴 내에서는 표현식 삽입을 통해 문자열 연산자 보다 가독성 좋고 간편하게 문자를 조합할 수 있다.

          3. 표현식을 삽입하려면 ${} 으로 표현식을 감싼다 , 이때 표현식의 평가 결과가 문자열이 아니더라도 문자열로 타입이 강제로 변환되어 삽입된다.

        3. 테그드 템플릿 (tagged template)

        4. 템플릿 리터럴은 백틱을 사용해 표현한다.

  2. 불리언(boolean): true or false

    1. undefined: var 키워드로 선언을 했지만 값을 할당하지 않았을 때 변수에 암묵적으로 할당 되는 값

    2. null:값이 없다는 것을 의도적으로 명시할 때 사용하는 값

    3. Symbol : ES6 에서 추가된 7번째 타입

  3. 객체 타입 (object / reference type): 객체, 함수, 배열 등

    1. 자바스크립트는 객체 기반의 언어이며 자바스크립트를 이루고 있는 거의 모든 것이 객체이고 원시타입의 6가지를 제외 하면 나머지는 모두 객체 타입이다.

데이터 타입의 필요성

  1. 변수에 할당되는 값의 데이터 타입에 따라 확보해야 할 메모리 공간의 크기가 결정된다
  2. 값을 저장할 때 확보해야 하는 메모리 공간의 크기를 결정하기 위해
    1. ECMAscript 사양에 숫자 타입은 배정밀도 64비트 부동소수점 형식을 사용한다고 명시되어 있고 부동소수점 형식은 8바이트로 숫자를 표현하므로 이 책에서는 숫자 값의 크기를 8바이트로 설명한다
  3. 값을 참조할 때 한 번에 읽어 들여야 할 메모리 공간의 크기를 결정하기 위해
  4. 메모리에서 읽어 들인 2진수를 어떻게 해석할지 결정하기 위해

동적 타이핑

동적 타입 언어와 정적 타입 언어

  1. 정적 타입 언어
    1. 정적 타입 언어는 변수의 타입을 변경할 수 없으며, 변수에 선언한 타입에 맞는 값만 할당할 수 있다
    2. 정적 타입 언어는 컴파일 시점에 타입체크(선언한 타입에 맞는 값을 할당했는지)를 수행한다, 만약 타입체크를 통과하지 못한다면 에러를 발생시키고 프로그램의 실행을 막는다
  2. 동적 타입 언어
    1. 정정 타입 언어와는 다르게 변수를 선언할 때 타입을 선언하지 않는다.
    2. 동적 타입 언어는 어떠한 데이터 타입의 값이라도 자유롭게 할당할 수 있다.
    3. 자바스크립트 변수는 선언이 아닌 할당에 의해 타입이 결정 된다

동적 타입 언어와 변수

  1. 모든 소프트웨어 아키텍처에는 트레이드오프(trade-off)가 존재하며, 모든 애플리케이션에 적합한 은 탄환(silver bullet)은 없듯이 동적 타입 언어 또한 구조적이 단점이 있다
    1. 트레이드오프(trade-off)
      1. 두 개의 정책 목표 가운데 하나를 달성하려고 하면 다른 목표의 달성이 늦어지거나 희생되는 모순적 관계를 의미한다 예를 들어 실업률을 줄이면 물가가 상승하고, 물가를 안정시키면 실업률이 높아진다
    2. 은 탄환(silver bullet)
      1. 고질적인 문제를 단번에 해결할 수 있는 명쾌한 해결책
    3. 동적 타입 언어의 변수는 값을 확인하기 전에는 타입을 확신할 수 없다.
    4. 더욱이 자바스크립트는 개발자의 의도와는 상관없이 엔진에 의해 암묵적으로 타입이 자동으로 변환되기도 한다,이와 같은 이유 때문에 동적 타입 언어는 유연성은 높지만 신뢰성은 떨어진다.
    5. 이러한 이유로 안정적인 프로그램을 만들기 위해 변수를 사용하기 이전에 데이터 타입을 체크해야 하는 경우가 있는데 이는 매우 번거로울뿐더러 코드의 양도 증가한다.

변수를 사용할 때 주의사항

  1. 변수는 꼭 필요한 경우에 한해 제한적으로 사용한다, 변수 값은 재할당에 의해 언제든지 변경될 수 있다 이로인해 타입을 잘못 예측해 오류가 발생할 가능성이 크다,
  2. 변수의 유효 범위(스코프)는 최대한 좁게 만들어 변수의 부작용을 억제해야 한다 .
  3. 전역 변수는 최대한 사용하지 않도록 한다, 어디서든지 참조와 변경이 가능한 전역변수는 오류가 발생할 경우 원인을 특정하기 어렵게 만든다
  4. 변수 보다는 상수를 사용해 값의 변경을 억제한다 .
  5. 존재 이유를 파악할 수 있는 변수의 이름을 지어줘야 한다 .

연산자(operator)

  1. 연산자는 하나 이상의 표현식을 대상으로 산술, 할당, 비교, 논리, 타입, 지수 연산 등을 수행해 하나의 값을 만든다 .

  2. 연산의 대상을 피연산자 라고한다.

  1. 피연산자는 연산의 대상이 되어야 하므로 값으로 평가할 수 있어야한다 연산자는 값으로 평가된 피연산자를 연산해 새로운 값을 만든다.

  2. 산술 연산자 (산술 연산이 불가능한 경우 NaN을 반환한다)

    1. 이항 산술 연산자

      1. 2개의 피연산자를 산술하여 숫자 값을 만들며 덧셈,뺄셈,곱셈,나눗셈 등이 있다.

    2. 단항 산술 연산자

      1. 1개의 피연산자를 산술 연산하여 숫자 값을 만든다.

      2. ++(증가), –(감소), +(어떠한 효과도 없다), -(양수를 음수로, 음수를 양수로 반전한 값을 반환한다.)

      4. 증가/감소(++/–)연산자는 위치에 의미가 있다

        1. 피연산자 앞에 위치한 증가/감소 연산자는 값을 증가/감소 시킨후 다른 연산을 진행한다

        2. 피연산자 뒤에 위치한 증가/감소 연산자는먼저 다른 연산을 수행한 후, 값을 증가/감소 시킨다.

  3. +단항 연산자는 피연산자에 어떠한 효과도 없다, 음수를 양수로 반환하지도 않는다 단, 숫자 타입이 아닌 피연산자에 사용하면 피연산자를 숫자 타입으로 변환하여 반환한다, 피연산자를 변경하는 것은 아니고 숫자 타입으로 변환한 값을 생성해서 반환한다

  4. -단항 연산자는 피연산자의 부호를 반전, +단항 연산자와 마찬가지로 숫자타입이아닌 피연산자를 숫자 타입으로 변환하여 반환, 부호를 반전한 값을 생성해 반환한다 .

      1. 문자열 연결 연산자

        1. +연산자는 피연산자 중 하나 이상이 문자열인 경우 문자열 연결 연산자로 동작한다.

할당 연결자

  1. 우항에 있는 피연산자의 평가 결과를 좌항에 있는 변수에 할당한다. 할당 연산자는 좌항의 변수에 값을 할당하므로 변수 값이 변하는 부수 효과가 있다

  2. 표현식은 값으로 평가될 수 있는 문, 문에는 표현식과 표현식이 아닌 문

    1. 할당문은 값으로 평가되는 표현식인 문으로서 할당된 값으로 평가된다 .

비교 연산자

  1. 좌항과 우항의 피연산자를 비교한 다음 그 결과를 불리언 값으로 반환 한다 ,
  2. 비교연산자는 if문이나 for 문과 같은 제어문의 조건식에서 주로 사용한다.
    1. x===y , x와 y의 값과 타입이 같음
    2. !== , x와 y의 값과 타입이 다름

삼항 조건 연산자

  1. 조건식의 평가 결과에 따라 반환할 값을 결정한다
  2. 문법
    1. 조건식 ? 조건식이 true 일때 반환할 값 : false일 때 반환할 값
    2. var result = score >= 60 ? ‘pass’ : ‘fail’
    3. true로 평가되면 pass 반환, false일 시 fail 반환
