    1. 객체 리터럴의 프로퍼티는 프로퍼티 키와 값으로 구성된다, 프로퍼티 값은 변수에 할당된 값, 즉 식별자 표현식일 수도 있다,.
    3. ES6에서는 프로퍼티 값으로 변수를 사용하는 경우 변수 이름과 프로퍼티 키가 동일한 이름일 때, 프로퍼티 키를 생략(property shorthand)할 수 있다, 이때 프로퍼티 키는 변수 이름으로 자동 생성 된다.

  2. 계산된 프로퍼티 이름

    1. 문자열 또는 문자열로 타입 변환할 수 있는 값으로 평가되는 표현식을 사용해 프로퍼티 키를 동적으로 생성할 수도 있다 단, 프로퍼티 키로 사용할 표현식을 대괄호로 묶어야 한다 이를 계산된 프로퍼티 이름(computed property name)이라고 한다.
    2. ES5에서는 객체리터럴 외부에서, ES6에서는 내부에서 계산된 프로퍼티 이름으로 프로퍼티 키를 동적 생성할 수 있다,

  3. 메서드 축약 표현

    1. ES5에서 메서드를 정의하려면 프로퍼티 값으로 함수를 할당한다.
    2. ES6에서 메서드를 정의할 때 function 키워드를 생략한 축약 표현을 사용할 수 있다.
    4. ES6의 메서드 축약 표현으로 정의한 메서드는 프로퍼티에 할당한 함수와 다르게 동작한다 .

원시값과 객체의 비교

  1. 데이터 타입에서 살펴 보았듯이 자바스크립트가 제공하는 7가지 데이터 타입은 크게 원시타입(primitive type)과 객체(object / reference)타입으로 구분할 수 있다
  2. 원시타입과 객체 타입은 근본적으로 다른데 크게 세가지 측면에서 다르다
    1. 원시 타입의 값, 즉 원시값은 변경 불가능한 값(immutable value)이다, 객체는 변경 가능한 값(mutable value)이다.
    2. 원시값을 변수에 할당하면 변수(확보된 메모리 공간)에는 실제 값이 저장된다 이에 비해 객체를 변수에 할당하면 변수(확보된 메모리 공간)에는 참조값이 저장된다 .
    3. 원시값을 갖는 변수를 다른 변수에 할당하면 원본의 원시값이 복사되어 전달된다, 이를 값에 의한 전달(pass by value)이라고 한다 이에 비해 객체를 가리키는 변수를 다른 변수에 할당하면 원본의 참조값이 복사되어 전달된다. 이를 **참조에 의한 전달 (pass by reference)**라고 한다.

원시값

  1. 원시 타입(prinitive type)의 값 즉 원시값은 변경 불가능한 값 (immutable value)이고 읽기 전용값으로서 변경할 수 없다.
  2. 변수는 하나의 값을 저장하기 위해 확보한 메모리 공간 자체 또는 그 메모리 공간을 식별하기 위한 이름이고 ,
  3. 값은 변수에 저장된 데이터로서 표현식이 평가되어 생성된 결과를 말한다
  4. 변경 불가능하다는것은 변수가 아니라 값에 대한 진술이다

객체 리터럴

객체란?

  1. 자바스크립트는 객체(object) 기반의 프로그래밍 언어이며, 자바스크립트를 구성하는 거의 “모든것”이 객체다
  2. 원시값을 제외한 나머지 값들(함수, 배열, 정규표현식 등)은 모두 객체다.
  3. 원시 타입은 단 하나의 값만을 나타내지만 객체 타입(object/ reference type)은 다양한 타입의 값(원시값 또는 다른 객체)을 하나의 단위로 구성한 복합적인 자료구조(data structure)다.
  4. 원시값은 변경 불가능한 값(immutable value)지만 객체 타입의 값, 즉 객체는 변경 가능한 값(mutable value)이다.
  5. 객체는 0개 이상의 프로퍼티로 구성된 집합이며 , 프로퍼티는 키(key)와 값(value)으로 구성된다.
  6. 객체는 프로퍼티와 메서드로 구성된 집합체 이다

프로퍼티(property)

  1. 프로퍼티 : 객체의 상태를 나타내는 값
  2. 메서드 : 프로퍼티(상태 데이터)를 참조하고 조작할 수 있는 동장(behavior)
  3. 자바스크립트에서 사용할 수 있는 모든 값은 프로퍼티 값이 될 수 있다.
  4. 자바스크립트에서 함수는 일급 객체이므로 값으로 취급할 수 있다
  5. 프로퍼티 값이 함수일 경우 일반 함수와 구분하기 위해 메서드(mathod)라고 부른다.
  6. 이처럼 객체는 객체의 상태를 나타내는 값(프로퍼티)와 프로퍼티를 참조하고 조작할 수 있는 동작(메서드)를 모두 포함할 수 있기 때문에 상태와 동작을 하나의 단위로 구조화 할 수 있어 유용하다.
  7. 객체는 프로퍼티(property)들의 집합이며 프로퍼티는 키와 값으로 구성된다.
  8. 프로퍼티를 나열할 때는 쉼표(,)로 구분한다. 일반적으로 마지막 프로퍼티 뒤에는 쉼표를 사용하지 않으나 사용해도 좋다.
    1. Var person = {name:’Lee’, age:20};
    2. 프로퍼티의 키는 name, 값은 Lee
    3. 프로퍼티의 키는 age, 값은 20
  9. 프로퍼티 키 : 빈문자열을 포함하는 모든 문자열 또는 심벌 값.
  10. 프로퍼티 값 : 자바스크립트에서 사용할 수 있는 모든값.

객체 리터럴에 의한 객체 생성

  1. 클래스 기반 객체지향 언어는 클래스를 사전에 정의하고 필요한 시점에 new연산자와 함께 생성자(constructor)를 호출하여 인스턴스를 생성하는 방식으로 객체를 생성한다.
    1. 인스턴스(instance)
    2. 인스턴스란 클래스에 의해 생성되어 메모리에 저장된 실체를 말한다, 객체지향 프로그래밍에서 객체는 클래스와 인스턴스를 포함한 개념이다 클래스는 인스턴스를 생성하기 위한 템플릿 역할을 한다, 인스턴스는 객체가 메모리에 저장되어 실제로 존재하는 것에 초점을 맞춘 용어다
  2. 자바스크립트는 프로토타입 기반 객체지향 언어로서 클래스 기반 객체지향 언어와는 달리 다양한 객체 생성 방법을 지원한다
    1. 객체 리터럴
    2. Object 생성자 함수
    3. 생성자 함수
    4. object.create 메서드
    5. 클래스(ES6)
  3. 이중에 가장 일반적으로 간단한 방법은 객체 리터럴을 사용하는 방법이다 , 리터럴은 사람이 이해할 수 있는 문자 또는 약속된 기호를 사용하여 값을 생성하는 표기법을 말한다, 객체 리터럴은 객체를 생성하기 위한 표기법이다.
  4. 변수에 할당이 이루어지는 시점에 자바스크립트 엔진은 객체 리터럴을 해석해 객체를 생성한다 .
  6. 만약 중괄호 내에 프로퍼티를 정의하지 않으면 빈 객체가 생성된다.
  7. 객체 리터럴의 중괄호는 코드 블록을 의미하지 않는다는 것에 주의하자, 코드 블록의 닫는 중괄호 뒤에는 세미콜론을 붙이지 않는다, 하지만 객체 리터럴은 값으로 평가되는 표현식이다 따라서 객체 리터럴의 닫는 중괄호 뒤에는 세미콜론을 붙인다.

타입변환

  1. 개발자가 의도적으로 값의 타입을 변환하는 것을 **명시적 타입변환(explicit coercion)**또는 **타입 캐스팅(type casting)**이라고 한다
  2. 개발자의 의도와는 상관 없이 표현식을 평가하는 도중에 자바크스크립트 엔진에 이해 암묵적으로 타입이 자동 변환 되기도 한다 .
  3. 이를 암묵적 타입변환(implicit coercion) 타입 강제 변환 (type coercio)이라고 한다 .
  4. 명시적 타입 변환이나 암묵적 타입 변환 기준이 기존 원시값을 직접 변경하는 것은 아니다
  5. 원시값은 변경 불가능한 값이므로 변경할 수 없다, 타입 변환이란 기존 원시값을 사용해 다른 타입의 새로운 원시값을 생성하는 것이다.

문자열 타입으로 변환

  1. +산술 연산자는 피연산자중 하나 이상이 문자열일 경우 문자열 연결 연산자로 동작한다
    1. 1 + ‘2’ // -> “12” 숫자열 1을 문자열로 타입변환하여서 문자열 12출력

숫자 타입으로 변환

  1. -,*,/ 산술 연산자의 역할은 숫자 값을 만드는 것이다 따라서 산술 연산자의 피연산자는 코드 문맥상 숫자 타입이여야 한다
  2. 그렇기 때문에 엔진은 피연산자 중에서 숫자 타입이 아닌 피연산자를 숫자타입으로 암묵적 타입변환을 한다
  3. 이 때 숫자 타입으로 변호한 될수 없는 경우는 산술연산을 할 수 없기때문에 NaN을 반환한다.
  4. 비교 연산자의 역할은 불리언 값을 만드는 것이다. 크기를 비교하는 비교연산자는 모두 숫자 타입이어야 한다
  5. 이 역시 숫자 타입이 아닌 피연산자를 숫자타입으로 암묵적 타입변환을 한다 .
  6. 비어있는 문자열,배열,null,false는 0, true는 1로 변환된다. 객체와 빈 배열이 아닌 배열, undefined는 NaN이 된다.

불리언 타입으로 변환

  1. if문이나 for문과 같은 제어문 또는 삼항 조건 연산자의 조건식은 불리언 값, 즉 논리적 참/거짓으로 평가되어야 하는 표현식이다.
  2. 자바스크립트 엔진은 조건식의 평가 결과를 불리언 타입으로 암묵적 타입 변환한다 .

단축 평가

논리 연산자를 사용한 단축 평가

  1. || (논리합) 연산자와 &&(논리곱) 연산자 표현식의 평가 결과는 불리언 값이 아닐 수도 있다.
  2. 두 연산자는 언제나 2개의 피연산자 중 어느 한쪽으로 평가된다 .
  3. 논리곱
    1. ‘cat’ && ‘dog’ // >> dog반환
    2. &&(논리곱) 연산자는 두 개의 피연산자가 모두 true로 평가될 때 true를 반환한다 . 좌항에서 우항으로 평가가 진행된다.
    3. 첫번째 ‘cat’은 thurthy 값이므로 ture로 평가된다. 하지만 이 시점까지는 위 표현식을 평가할 수 없다
    4. 두번째 피연산자가 위 논리곱 연산자 표현식의 평가 결과를 결정한다 . 이때 논리곱 연산자는 논리 연산의 결과를 결정하는 두 번째 피연산자, 즉 문자열’Dog’를 그대로 반환한다 .
  4. 논리합
    1. ‘cat’ || ‘dog’ // >> cat 반환
      1. 논리합 연산자는 두 개의 피연산자 중 하나만 true여도 true를 반환한다, 마찬가지로 좌항에서 우항으로 평가 된다.
      2. 첫번째 피연산자 ‘cat’은 thurhy값이 므로 true로 평가된다. 이때 논리합 연산자는 논리 연산의 결과를 결정한 첫 번째 피연산자, 즉 문자열 ‘cat’을 그대로 반환한다.
  5. 이처럼 논리 연산의 결과를 결정하는 피연산자를 타입 변환하지 않고 그대로 반환한다, 이를 단축평가라고 한다
  6. 단축 평가는 표현식을 평가하는 도중에 평가 결과가 확정된 경우 나머지 평가 과정을 생략하는 것을 말한다.
  7. 어떤 조건이 Thurthy값일 때 무언가를 해야 한다면 논리곱 연산자 표현식으로 if문을 대체할 수 있다.
  8. 반대로 어떤 조건이 Falsy값일 때 무언가를 해야 한다면 논리합 연산자 표현식으로 if문을 대체할 수 있다.

옵셔널 체이닝 연산자(optional chaining)

  1. ES11 에서 도입된 옵셔널 체이닝 연산자 ?.는 좌항의 피연산자가 null또는 undefined인 경우 undefined 반환하고, 그렇지 않으면 우항의 프로퍼티 참조를 이어간다 .
  2. 논리 연산자 &&는 좌항 피연산자가 false로 평가되는 falsy값(false, undefined, null, 0 -0, NaN, “”)이면 좌항 피연산자를 그대로 반환한다
  3. flasy값인 0이나 “인 경우도 마찬가지다 하지만 0이나 “은 객체로 평가될 때도 있다.

null 병합 연산자

  1. ES11 에서 도입된 null병합 연산자 ??는 좌항의 피연산자가 null 또는 undefined인 경우 우항의 피연산자를 반환하고 , 그렇지 않으면 좌항의 피연산자를 반환한다.
  2. Null 병합 연산자 ??는 변수에 기본값을 설정할 때 유용하다 .
  3. null 병합 연산자가 도입되기 전에는 논리 연산자 ||를 사용한 단축 평가를 통해 변수에 기본 값을 설정했다.
  4. 논리 연산자 ||를 사용한 단축 평가의 경우 좌항의 피연산자가 false로 평가되는 falsy값 (false, undefined, null, 0 ,-0, NaN, “)이면 우항의 피연산자를 반환한다, 만약 falsy 값인 0 이나 “도 기본값으로서 유효하다면 예기치 않은 동작이 발생할 수 있다.
  5. 하지만 null 병합 연산자 ??는 좌항의 피연산자가 false로 평가되는 Falsy 값(false, undefined, null, 0, -0, NaN, ‘‘)이라도 null 또는 undefined가 아니면 좌항의 피연산자를 그대로 반환한다.

동적 타입 언어와 정적 타입 언어

  1. 정적 타입 언어
    1. 정적 타입 언어는 변수의 타입을 변경할 수 없으며, 변수에 선언한 타입에 맞는 값만 할당할 수 있다
    2. 정적 타입 언어는 컴파일 시점에 타입체크(선언한 타입에 맞는 값을 할당했는지)를 수행한다, 만약 타입체크를 통과하지 못한다면 에러를 발생시키고 프로그램의 실행을 막는다
  2. 동적 타입 언어
    1. 정정 타입 언어와는 다르게 변수를 선언할 때 타입을 선언하지 않는다.
    2. 동적 타입 언어는 어떠한 데이터 타입의 값이라도 자유롭게 할당할 수 있다.
    3. 자바스크립트 변수는 선언이 아닌 할당에 의해 타입이 결정 된다

동적 타입 언어와 변수

  1. 모든 소프트웨어 아키텍처에는 트레이드오프(trade-off)가 존재하며, 모든 애플리케이션에 적합한 은 탄환(silver bullet)은 없듯이 동적 타입 언어 또한 구조적이 단점이 있다
    1. 트레이드오프(trade-off)
      1. 두 개의 정책 목표 가운데 하나를 달성하려고 하면 다른 목표의 달성이 늦어지거나 희생되는 모순적 관계를 의미한다 예를 들어 실업률을 줄이면 물가가 상승하고, 물가를 안정시키면 실업률이 높아진다
    2. 은 탄환(silver bullet)
      1. 고질적인 문제를 단번에 해결할 수 있는 명쾌한 해결책
    3. 동적 타입 언어의 변수는 값을 확인하기 전에는 타입을 확신할 수 없다.
    4. 더욱이 자바스크립트는 개발자의 의도와는 상관없이 엔진에 의해 암묵적으로 타입이 자동으로 변환되기도 한다,이와 같은 이유 때문에 동적 타입 언어는 유연성은 높지만 신뢰성은 떨어진다.
    5. 이러한 이유로 안정적인 프로그램을 만들기 위해 변수를 사용하기 이전에 데이터 타입을 체크해야 하는 경우가 있는데 이는 매우 번거로울뿐더러 코드의 양도 증가한다.

변수를 사용할 때 주의사항

  1. 변수는 꼭 필요한 경우에 한해 제한적으로 사용한다, 변수 값은 재할당에 의해 언제든지 변경될 수 있다 이로인해 타입을 잘못 예측해 오류가 발생할 가능성이 크다,
  2. 변수의 유효 범위(스코프)는 최대한 좁게 만들어 변수의 부작용을 억제해야 한다 .
  3. 전역 변수는 최대한 사용하지 않도록 한다, 어디서든지 참조와 변경이 가능한 전역변수는 오류가 발생할 경우 원인을 특정하기 어렵게 만든다
  4. 변수 보다는 상수를 사용해 값의 변경을 억제한다 .
  5. 존재 이유를 파악할 수 있는 변수의 이름을 지어줘야 한다 .

연산자(operator)

  1. 연산자는 하나 이상의 표현식을 대상으로 산술, 할당, 비교, 논리, 타입, 지수 연산 등을 수행해 하나의 값을 만든다 .

  2. 연산의 대상을 피연산자 라고한다.

  1. 피연산자는 연산의 대상이 되어야 하므로 값으로 평가할 수 있어야한다 연산자는 값으로 평가된 피연산자를 연산해 새로운 값을 만든다.

  2. 산술 연산자 (산술 연산이 불가능한 경우 NaN을 반환한다)

      1. 2개의 피연산자를 산술하여 숫자 값을 만들며 덧셈,뺄셈,곱셈,나눗셈 등이 있다.

    2. 단항 산술 연산자

      1. 1개의 피연산자를 산술 연산하여 숫자 값을 만든다.

      2. ++(증가), –(감소), +(어떠한 효과도 없다), -(양수를 음수로, 음수를 양수로 반전한 값을 반환한다.)

변수에 선언과 초기화, undefined가 할당되는 시점

  1. 어제 공부한내용은 변수의 선언을 함과 동시에 초기화가 진행되면서 undefined값이 할당된다고 나와있지만 사실은 변수를 선언하고 그 할당된 값을 참조하는 순간에 undefined가 할당된다 , 처음부터 이렇게 설명하면 초보자가 접근하기에 어려울 수 있기 때문에 많은 책에 이부분이 빠져있다.

키워드 ( keyword)

  1. 자바스크립트 엔진의 명령어를 의미한다
  2. 변수나 식별자는 실행컨텍스트에서 등록된다
    1. 실행컨텍스트란 자바스크립트 엔진이 소스코드를 평가하고 실행하는 환경과 그 결과를 관리하는 영역이다
    2. 실행컨텍스트를 통해서 자바스크립트 엔진은 식별자와 스코프를 관리한다 .

var,let,const 초기화를 진행하는 이유와 키워드의 선언,초기화 순서

  1. 이전 애플리케이션의 메모리 값이 존재할 수 있기 때문에 초기화를 진행해준다 .
  2. var키워드의 경우 런타임이전에 선언과 동시에 초기화를 진행하고 값에 접근하는 순간 undefined가 할당된다
  3. Let의 경우 런타임 이전에 선언단계가 시작하고 이후에 초기화가 진행되면서 스코프의 시작지점부 초기화 단계 시작지점 까지 일시적 사각지대가 생겨난다
    2. 위와 같은 구조이기 때문에 var와 다르게 런타임 이전에 변수를 참조할 경우에 ReferenceError가 뜨면서 호이스팅 현상이 발생하지 않는 것처럼 보인다
  4. const의 경우 변경할수 없는 값인 상수로 변수의 선언과 동시에 값의 할당이 이루어지기 때문에 초기화단계가 필요없다.

값을 재할당 할 시 메모리의 변화

  1. 변수 선언 시 초기화를 진행하면서 존재하던 undefined에 새로운 값을 넣어주었을 때
  2. undefined가 있던 메모리셀이 초기화되면서 새로운 값이 들어가지 않고
  3. 새로운 메모리 셀에 새로운 값이 추가되고 변수는 새로운 값이 추가된 메모리셀의 공간의 주소를 기억한다
  4. 그리고 이전에 존재했던 undefined는 추적하는 변수가 존재하지 않기 때문에 가비지 콜렉터에 의해 메모리가 해제된다
    1. 단 해제되는 시점은 알 수 없다 그렇기 때문에 초기화가 진행되야하고 초기화를 하지 않을 경우 해제되지 않은 메모리에 접근할 수 있게된다

가비지 콜렉터 (Garbage collector)

  1. 가비지 콜렉터란 사용되지 않는 메모리를 주기적으로 검사하여 해제 시켜주는 기능이다.

언매니지드 언어와 매니지드 언어

  1. C같은 언어는 메모리를 직접적으로 관리 할 수 있기 때문에 언매니지드 언어로써 가비지 콜렉터가 존재하지 않는다.
    1. 개발자의 역량에 따라 최고의 효율을 낼 수 있지만 반대의 경우 치명적인 오류가 생길 수 있다.
  2. 우리가 사용하는 javascript언어는 메모리를 직접적으로 관리 할 수 없기 때문에 매니지드 언어로써 가비지 콜렉터가 존재한다.
    1. 성능면에서 언매니지드 언어보다 불편하지만 메모리를 직접 관리하지 않는다는 점에서 편리하다.

키워드(keyword)

  1. 키워드는 자바스크립트 코드를 해석하고 실행하는 자바스크립트 엔진이 수행할 동작을 규정한 일종의 명령어다. 자바스크립트 엔진은 키워드를 만나면 자신이 수행해야 할 약속된 동작을 수행한다, 예를 들어, var 키워드를 만나면 자바스크립트 엔진은 뒤에 오는 변수 이름으로 새로운 변수를 선언한다 .
  2. 변수를 선언한 후 변수에 값을 할당하지 않을 경우 빈 공간으로 생각할 수 있으나 자바스크립트 엔진에 의해 undefined라는 값이 암묵적으로 할당되어 초기화 된다 .
  3. 변수 이름을 비롯한 모든 식별자는 실행 컨텍스트에 등록된다,
    1. 실행 컨텍스트는 자바스크립트 엔진이 소스코드를 평가하고 실행하기 위해 필요한 환경을 제공하고 코드의 실행 결과를 실제로 관리하는 영역이다
    2. 자바스크립트 엔진은 실행 컨텍스트를 통해 식별자와 스코프를 관리한다.
  4. var 키워드를 사용한 변수 선언은 선언,초기화 단계가 동시에 진행된다.
    1. 일반적으로 초기화(initialization)란 변수가 선언된 이후 최초로 값을 할당하는 것을 말한다 .
    2. var 키워드로 선언한 변수는 undefined로 암묵적인 초기화가 자동수행 된다.
  5. 만약 초기화 단계를 거치지 않으면 확보된 메모리 공간에는 이전에 다른 애플리케이션이 사용했던 값이 남아 있을 수 있다. 이러한 값을 쓰레기 값(garbage value)이라고 한다
  6. 따라서 메모리 공간을 확보한 다음 값을 할당하지 않은 상태에서 곧바로 변수 값을 참조하면 쓰레기 값이 나올 수 있다, 자바스크립트의 var 키워드는 암묵적으로 초기화를 수행하므로 이러한 위험으로부터 안전하다.
    1. var의 경우 선언과 초기화 단계가 동시에 진행되지만 let은 선언단계와 초기화 단계가 분리되어 진행된다
    2. const의 경우는 재할당이 금지되기 때문에 선언함과 동시에 값이 할당되어야 한다 그렇기 때문에 초기화가 진행되지 않는다.

변수 선언의 실행 시점과 변수 호이스팅

  2. 위 예제는 변수 선언문보다 변수를 참조하는 코드가 앞에 있다,자바스크립트는 인터프리터 언어로써 한 줄씩 순차적으로 실행되므로 위의 예제와 같이 실행될경우 선언되기 이전에 코드를 실행했기 때문에 ReferenceError가 발생할 것 처럼 보이지만 실제론 undefined가 출력된다.

  3. 그 이유는 변수 선언이 소스코드가 한 줄씩 순차적으로 실행되는 시점이 아니라 그 이전 단계에서 먼저 실행되기 때문이다.

  4. 자바스크립트 엔진은 소스코드를 한 줄씩 순차적으로 실행하기 전에 소스코드의 평가 과정을 거치면서 실행하기 위한 준비를한다

  5. 이때 자바스크립트 엔진은 변수 선언을 포함한 모든 선언문 (변수 선언문,함수 선언문 등)을 소스 코드에서 찾아내 먼저 실행한다.

  6. 즉 변수 선언은 어디에 있든 상관 없이 다른 코드보다 먼저 실행한다

  7. 이처럼 변수 선언문이 코드의 선두로 끌어 올려진 것처럼 동작하는 자바스크립트 고유의 특징을 변수 호이스팅(Variable hoisting) 이라고 한다.

  8. *사실 호이스팅은 변수 선언 뿐만이 아니라 var, let, const, function, function*, class 키워드를 사용해서 선언하는 모든 식별자(변수, 함수, 클래스 등)는 호이스팅된다. 모든 선언문은 런타임 이전 단계에서 먼저 실행되기 때문이다.*

값의 재할당

  2. 재할당은 변수에 저장된 값을 다른 값으로 변경한다, 만약 값을 재할당 할 수 없어서 변수에 저장된 값을 변경할 수 없다면 변수가 아니라 상수(constant)라고 한다

    1. Const 키워드
      1. ES6에서 도입된 const키워드를 사용해 선언한 변수는 재할당이 급지된다, 하지만 const키워드는 반드시 상수만을 위해 사용하지는 않는다.

  3. 변수에 값을 재할당 하게 되면 이전값이 저장되어 있던 메모리 공간을 지우고 그 메모리 공간에 재할당 된 값을 저장하는것이 아니라 새로운 메모리 공간을 확보하고 그 메모리 공간에 숫자 값을 저장한다.

    2. 현재 score변수의 값은 90이다, 변수의 이전 값인 undefined와 80은 어떤 변수도 값으로 갖고 있지 않고 어떤 식별자와도 연결되어 있지 않기 때문에 더이상 필요하지 않은 값이란 의미다, 이러한 불필요한 값들은 가비지 콜렉터에 의해서 자동으로 해제 된다, 단 메모리에서 언제 해제될지는 예측할 수 없다.
    3. 가비지 콜렉터(garbage collector)
      1. 가비지 콜렉터는 애플리케이션이 할당한 메모리 공간을 주기적으로 검사하여 더이상 사용되고 있지 않은 메모리를 해제(relwase)하는 기능을 말한다.
      2. 자바스크립트는 가비지 콜렉터를 내장하고 있는 매니지드 언어로서 가비지 콜렉터를 통해 메모리 누수(memory leak)를 방지한다.
      3. 언매니지드 언어와 매니지드 언어
        1. C 언어 같은 언매니지드 언어는 개발자가 명시적으로 메모리를 할당하고 해제하기 위해 malloc()과 free() 같은 저수준(low-level) 메모리 제어 기능을 제공한다. 언매니지드 언어는 메모리 제어를 개발자가 주도할 수 있으므로 개발자의 역량에 따라 최적의 성능을 확보할 수 있지만 그 반대의 경우 치명적 오류를 생산할 가능성도 있다.
        2. 자바스크립트 같은 매니지드 언어는 메모리의 할당 및 해제를 위한 메모리 관리 기능을 언어 차원에서 담당하고 개발자의 직접적인 메모리 제어를 허용하지 않는다. 즉, 개발자가 명시적으로 메모리를 할당하고 해제할 수 없다. 더 이상 사용하지 않는 메모리의 해제는 가비지 콜렉터가 수행하며, 이 또한 개발자가 관여할 수 없다. 매니지드 언어는 개발자의 역량에 의존하는 부분이 상대적으로 작아져 어느 정도 일정한 생산성을 확보할 수 있다는 장점이 있지만 성능 면에서 어느 정도의 손실은 감수할 수밖에 없다.

식별자의 네이밍 규칙

  1. 특수문자를 제외한 문자, 숫자, 언더스코어(_), 달러 기호($)를 포함할 수 있다.

  2. 단, 식별자는 특수문자를 제외한 문자, 언더스코어(_), 달러 기호($)로 시작해야 한다, 숫자로 시작할 수 없다.

  3. 예약어는 식별자로 사용할 수 없다

    1. 예약어(reserved word) = 프로그래밍 언어에서 사용되고 있거나 사용될 예정인 단어를 말한다.
      1. await break case catch class const continue debugger default delete do else enum export extends false finally for function if implements* import in Instanceof interface* let* new null package* private* protected* public* return super static* switch this throw true try typeof var void while with yield*

  4. 네이밍 컨벤션은 하나 이상의 영어 단어로 구성된 식별자를 만들 때 가독성 좋게 단어를 한눈에 구분하기 위해 규정한 명명규칙이다

    2. 가장 일반적인 방식은 변수나 함수의 이름에는 카멜케이스를 사용하고 생성자 함수, 클래스의 이름에는 파스칼 케이스를 사용하는 것이다.

    3. 자바스크립트에서는 일반적으로 변수나 함수의 이름에는 카멜 케이스를 사용하고. 생성자 함수, 클래스의 이름에는 파스칼 케이스를 사용한다.

표현식과 문

  1. 값

    1. 값(value)은 식(표현식,expression)이 평가(evaluate)되어 생성된 결과를 말한다. 평가란 식을 해석해서 값을 생성하거나 참조하는 것을 의미한다
    2. var sum = 10 + 20;
      1. 위 변수에 할당되는 것은 10+20이 아니라 10+20이 평가된 결과인 숫자 값 30이다.
    3. 값은 다양한 방법으로 생성할 수 있다, 가장 기본적인 방법은 리터럴을 사용하는 것이다.

  2. 리터럴

    1. 리터럴(literal)은 사람이 이해할 수 있는 문자 또는 약속된 기호를 사용해 값을 생성하는 표기 방식 (notaion)을 말한다.

    2. 숫자 리터럴 3은 단순한 아라비아 숫자가 아니라 숫자 리터럴이다, 사람이 이해할 수 있는 아라비아 숫자를 사용해 숫자 리터럴3을 코드에 기술하면 자바스크립트 엔진은 이를 평가해 숫자 값 3을 생성한다.

    4. 이처럼 리터럴은 사람이 이해할 수 있는 문자(아라비아 숫자, 알파벳, 한글 등) 또는 미리 약속된 기호로 표기한 코드다

    5. 자바스크립트 엔진은 코드가 실행되는 시점인 런타임에 리터럴을 평가해 값을 생성한다 즉, 리터럴은 값을 생성하기 위해 미리 약속한 표기법이라고 할 수 있다.

    6. 리터럴을 사용하면 다음과 같이 다양한 종류(data type)의 값을 생성할 수 있다.

  3. 표현식

    1. 표현식(expression)은 값으로 평가될 수 있는 문(statement)이다 즉, 표현식이 평가되면 새로운 값을 생성하거나 기존 값을 참조한다.
    2. 리터럴은 값으로 평가, 리터럴 = 표현식
    3. 이처럼 표현식은 리터럴, 식별자(변수, 함수 등의 이름), 연산자, 함수 호출 등의 조합으로 이뤄질 수 있다. 다음과 같이 다양한 표현식이 있지만 값으로 평가된다는 점에서 모두 동일하다. 즉, 값으로 평가되는 문은 모두 표현식이다.
    4. 표현식은 값으로 평가된다, 이때 표현식과 표현식이 평가된 값은 동등한 관계(동치equivalent)이다, 1+2 = 3에서 1+2와 3은 동치이다 따라서**표현식은 값처럼 사용할 수 있다**

  4. 문(statement)

    1. 프로그램을 구성하는 기본 단위이자 최소 실행 단위
    2. 문의 집합이 바로 프로그램이고, 문을 작성하고 순서에 맞게 나열하는 것이 프로그래밍이다 .
    3. 문은 여러 토큰으로 구성된다. 토큰(token)이란 문법적인 의미를 가지며, 문법적으로 더 이상 나눌 수 없는 코드의 기본 요소를 의미한다. 예를 들어, 키워드, 식별자, 연산자, 리터럴, 세미콜론(;)이나 마침표(.) 등의 특수 기호는 문법적인 의미를 가지며, 문법적으로 더 이상 나눌 수 없는 코드의 기본 요소이므로 모두 토큰이다.

  5. 위 예제의 숫자 값 10과 20은 메모리상의 임의의 위치(메모리 주소)에 기억되고 CPU는 이 값을 읽어 들여 연산을 수행한다. 연산 결과로 생성된 숫자 30도 메모리로 저장되고 저장되는 모든 값은 2진수로 저장된다

  6. 10 + 20이라는 연산을 했다는 것은 그 연산 결과가 필요하고 이를 사용해 무언가를 하겠다는 의도가 있었을 것이다, 연산 결과 30을 재사용 하고 싶다면 메모리 주소에 직접 접근하는 것 외에는 방법이 없다

    1. 하지만 메모리 주소를 통해 값에 직접 접근하는 것 은 치명적 오류를 발생시킬 가능성이 높은 매우 위험한 일이다
    2. 만약 실수로 운영체제가 사용하고 있는 값을 변경하면 시스템을 멈추게 하는 치명적인 오류가 발생할 수도 있다. 때문에 자바스크립트는 개발자의 직접적인 메모리 제어를 허용하지 않는다.
    3. 만약 자바스크립트가 개발자의 직접적인 메모리 제어를 허용 하더라도, 값이 저장될 메모리 주소는 코드가 실행될 때마다 값이 변경된다, 따라서 메모리 주소를 통해 값에 직접 접근하려는 시도는 올바른 방법이 아니다 .

  7. 변수(variable)은 하나의 값을 저장하기 위해 확보한 메모리 공간 자체 또는 그 메모리 공간을 식별하기 위해 붙인 이름을 말한다.

    1. 프로그래밍 언어에서 값을 저장하고 참조하는 매커니즘으로, 값의 위치를 가르키는 상징적인 이름이다.
    2. 변수는 프로그래밍 언어의 컴파일러 또는 인터프리터에 의해 값이 저장된 메모리 공간의 주소로 치환되어 실행된다. 따라서 개발자가 직접 메모리 주소를 통해 값을 저장하고 참조할 필요가 없고 변수를 통해 안전하게 값에 접근할 수 있다 .
      1. 컴파일러(compiler)란? = 특정 프로그래밍 언어로 쓰여있는 문서를 다른 프로그래밍 언어로 옮기는 프로그램.
      2. 인터프리터(interpreter)란? = 프로그래밍 언어의 소스 코드를 바로 실행하는 컴퓨터 프로그램 또는 환경.
  8. // 변수는 하나의 값을 저장하기 위한 수단
var userId = 1;
var userName = 'Lee';

// 객체나 배열 같은 자료구조를 사용하면 여러개의 값을 하나로 그룹화해서 하나의 값처럼 사용할 수 있다.
var user = {id:1, name: 'Lee'};

var users = {
  {id:1, name: 'Lee'}
    {id:2, name: 'Kim'}
}
    1. 객체(object)란? = 키(key)와 값(value)로 구성된 프로퍼티(property)들의 집합이다.
      1. 프로퍼티의 값으로 자바스크립트에서 사용할 수 있는 모든 값을 사용할 수 있다.
      2. 자바스크립트의 함수는 일급 객체이므로 값으로 취급할 수 있다, 따라서 프로퍼티 값으로 함수가 사용 가능하며 함수일 경우 일반 함수와 구분하기 위해 메소드라고 부른다
      3. 이와 같이 객체는 데이터를 의미하는 프로퍼티(property)와 데이터를 참조하고 조작할 수 있는 동작(bejavior)을 의미하는 메소드(method)로 구성된 집합 이다, 객체는 데이터(프로퍼티)와 그 데이터에 관려되는 동작(메소드)를 모두 포함할 수 있기 때문에 데이터와 동작을 하나의 단위로 구조화 할 수있어 유용하다.
    2. 자료구조란? = 컴퓨터 과학에서 효율적인 접근 및 수정을 가능케 하는 자료의 조직, 관리,저장을 의미한다.
      1. 데이터 값의 모임, 데이터 간의 관계, 데이터에 적용할 수 있는 함수나 명령을 의미
      2. 효과적으로 설계된 자료구조는 실행시간 혹은 메모리 용량과 같은 자원을 최소한으로 사용하면서 연산을 수행하도록 해준다.

  9. 변수에 값을 저장하는 것을 할당(assignment,대입,저장)이라고 하고, 변수에 저장된 값을 읽어들이는 것을 참조(reference)라고 한다.

식별자

  1. 변수 이름을 식별자(Identifier)라고도 한다.식별자란 어떤 값을 구별해서 식별할 수 있는 고유한 이름 을 말한다.
  2. 식별자는 값이 아니라 메모리 주소를 기억하고 있다.
  3. 메모리 상에 존재하는 어떤 값을 식별할 수 있는 이름은 모두 식별자 라고 부른다.
  4. 변수,함수, 클래스 등의 이름과 같은 식별자는 네이밍규칙을 준수해야하며 , 선언(declaration)에 의해 자바스크립트 엔진에 식별자의 존재를 알린다.

변수 선언

  1. 변수 선언(variable declaration)이란 변수를 생성하는 것을 말한다.
  2. 값을 저장하기 위한 메모리 공간을 확보(allocate)하고 변수 이름과 확보된 메모리 공간의 주소를 연결(name binding)해서 값을 저장할 수 있게 준비하는 것이다.
  3. 변수 선언에 의해 확보된 메모리 공간은 확보가 해제(release)되기 전까지는 누구도 확보된 메모리 공간을 사용할 수 없도록 보호된다.
  4. 변수를 선언 할 땐 var, let, const 키워드를 사용한다
    1. var의 단점중 가장 대표적인 것이 블록 레벨 스코프를 지원하지 않고 함수 레벨 스코프를 지원한다는 것이다.
