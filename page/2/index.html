블록문

  1. 0개 이상의 문을 중괄호로 묶은 것, 코드 블록 또는 블록.
  2. 자바스크립트는 블록문을 하나의 실행 단위로 취급한다, 제어문이나 함수를 정의할 때 사용하는 것이 일반적이다.
  3. {var foo = 10;} == 문의 끝에는 세미콜론을 붙이는 것이 일반적이지만 블록문은 자체 종결성을 갖기 때문에 붙이지 않는다.

조건문

  1. 조건문은 주어진 조건식의 평과 결과에 따라 코드 블럭의 실행을 결정한다
  2. 조건식은 불리언 값으로 평가될 수 있는 표현식이다. if..else문 , switch문

자바스크립트의 두 가지 조건문

  1. if..else문

    1. if문의 조건식은 불리언 값으로 평가 되어야 한다 .

    2. if문의 조건식이 불리언 값이 아닌 값으로 평가되면 자바스크립트 엔진에 의해 암묵적으로 불리언 값으로 강제 변한되어 실행할 코드 블록을 결정한다 .

    3. 조건식을 추가하여 조건에 따라 실행될 코드 블록을 늘리고 싶으면 else if문을 사용한다

    4. Else if 문과 else문은 옵션이다,if와 else문은 2번 이상 사용할 수 없지만 else if 문은 여러 번 사용할 수 있다.

    5. 대부분의 if..else문은 삼항 조건 연산자로 바꿔 쓸 수 있다.

      1. var x = 2;
var result;

if (x % 2) { // 2 % 2 는 0이다, 이때 0은 false로 암묵적 강제 변환 된다.
      result = '홀수';
} else {
  result = '짝수';
}

console.log(result); // 짝수

var x = 2;
var result = x % 2 ? '홀수' : '짝수';  // 0은 false로 취급된다
console.log(result); // true가 홀수 false가 짝수이기 때문에 2 % 2 = 0 이라서 짝수 반환

반복문

  1. 반복문은 조건식의 평가 결과가 참인 경우 코드 블록을 실행하고 조건식이 거짓일 때 까지 반복된다

  2. 세 가지 반복문인 for, while, do..while문을 제공한다.’

  3. **반복 횟수가 명확할땐 for, 명확하지 않을 때 while, **

  4. for문

    1. 조건식이 거짓으로 평가 될 때 까지 코드 블록을 반복 실행한다
    2. 실행 순서

  5. while문

    1. 주어진 평가 결과가 참이면 반복해서 실행한다

    2. 무한 루프에서 탈출 하기 위해서는 if 문으로 탈출 조건을 만들고 break문으로 코드 블록을 탈출한다 

    3. var count = 0;

// 무한루프
while (true) {
  console.log(count);
  count++;
  // count가 3이면 코드 블록을 탈출한다.
  if (count === 3) break;
} // 0 1 2

  6. do..while문

    1. 코드 블록을 먼저 실행하고 조건식을 평가한다, 따라서 무조건 한 번 이상 실행 된다 .

    1. 레블 문, 반복문, switch 문의 코드 블럭을 탈출한다 이외에 문에 사용하면 문법 에러가 발생한다 .

    1. 현 지점에서 코드 블록 실행을 중단하고 반복문의 증감식으로 실행 흐름을 이동 시킨다 .

타입변환

  1. 개발자가 의도적으로 값의 타입을 변환하는 것을 **명시적 타입변환(explicit coercion)**또는 **타입 캐스팅(type casting)**이라고 한다
  2. 개발자의 의도와는 상관 없이 표현식을 평가하는 도중에 자바크스크립트 엔진에 이해 암묵적으로 타입이 자동 변환 되기도 한다 .
  3. 이를 암묵적 타입변환(implicit coercion) 타입 강제 변환 (type coercio)이라고 한다 .
  4. 명시적 타입 변환이나 암묵적 타입 변환 기준이 기존 원시값을 직접 변경하는 것은 아니다
  5. 원시값은 변경 불가능한 값이므로 변경할 수 없다, 타입 변환이란 기존 원시값을 사용해 다른 타입의 새로운 원시값을 생성하는 것이다.

문자열 타입으로 변환

  1. +산술 연산자는 피연산자중 하나 이상이 문자열일 경우 문자열 연결 연산자로 동작한다
    1. 1 + ‘2’ // -> “12” 숫자열 1을 문자열로 타입변환하여서 문자열 12출력

숫자 타입으로 변환

  1. -,*,/ 산술 연산자의 역할은 숫자 값을 만드는 것이다 따라서 산술 연산자의 피연산자는 코드 문맥상 숫자 타입이여야 한다
  2. 그렇기 때문에 엔진은 피연산자 중에서 숫자 타입이 아닌 피연산자를 숫자타입으로 암묵적 타입변환을 한다
  3. 이 때 숫자 타입으로 변호한 될수 없는 경우는 산술연산을 할 수 없기때문에 NaN을 반환한다.
  4. 비교 연산자의 역할은 불리언 값을 만드는 것이다. 크기를 비교하는 비교연산자는 모두 숫자 타입이어야 한다
  5. 이 역시 숫자 타입이 아닌 피연산자를 숫자타입으로 암묵적 타입변환을 한다 .
  6. 비어있는 문자열,배열,null,false는 0, true는 1로 변환된다. 객체와 빈 배열이 아닌 배열, undefined는 NaN이 된다.

불리언 타입으로 변환

  1. if문이나 for문과 같은 제어문 또는 삼항 조건 연산자의 조건식은 불리언 값, 즉 논리적 참/거짓으로 평가되어야 하는 표현식이다.
  2. 자바스크립트 엔진은 조건식의 평가 결과를 불리언 타입으로 암묵적 타입 변환한다 .

단축 평가

논리 연산자를 사용한 단축 평가

  1. || (논리합) 연산자와 &&(논리곱) 연산자 표현식의 평가 결과는 불리언 값이 아닐 수도 있다.
  2. 두 연산자는 언제나 2개의 피연산자 중 어느 한쪽으로 평가된다 .
  3. 논리곱
    1. ‘cat’ && ‘dog’ // >> dog반환
    2. &&(논리곱) 연산자는 두 개의 피연산자가 모두 true로 평가될 때 true를 반환한다 . 좌항에서 우항으로 평가가 진행된다.
    3. 첫번째 ‘cat’은 thurthy 값이므로 ture로 평가된다. 하지만 이 시점까지는 위 표현식을 평가할 수 없다
    4. 두번째 피연산자가 위 논리곱 연산자 표현식의 평가 결과를 결정한다 . 이때 논리곱 연산자는 논리 연산의 결과를 결정하는 두 번째 피연산자, 즉 문자열’Dog’를 그대로 반환한다 .
  4. 논리합
    1. ‘cat’ || ‘dog’ // >> cat 반환
      1. 논리합 연산자는 두 개의 피연산자 중 하나만 true여도 true를 반환한다, 마찬가지로 좌항에서 우항으로 평가 된다.
      2. 첫번째 피연산자 ‘cat’은 thurhy값이 므로 true로 평가된다. 이때 논리합 연산자는 논리 연산의 결과를 결정한 첫 번째 피연산자, 즉 문자열 ‘cat’을 그대로 반환한다.
  5. 이처럼 논리 연산의 결과를 결정하는 피연산자를 타입 변환하지 않고 그대로 반환한다, 이를 단축평가라고 한다
  6. 단축 평가는 표현식을 평가하는 도중에 평가 결과가 확정된 경우 나머지 평가 과정을 생략하는 것을 말한다.
  7. 어떤 조건이 Thurthy값일 때 무언가를 해야 한다면 논리곱 연산자 표현식으로 if문을 대체할 수 있다.
  8. 반대로 어떤 조건이 Falsy값일 때 무언가를 해야 한다면 논리합 연산자 표현식으로 if문을 대체할 수 있다.

옵셔널 체이닝 연산자(optional chaining)

  1. ES11 에서 도입된 옵셔널 체이닝 연산자 ?.는 좌항의 피연산자가 null또는 undefined인 경우 undefined 반환하고, 그렇지 않으면 우항의 프로퍼티 참조를 이어간다 .
  2. 논리 연산자 &&는 좌항 피연산자가 false로 평가되는 falsy값(false, undefined, null, 0 -0, NaN, “”)이면 좌항 피연산자를 그대로 반환한다
  3. flasy값인 0이나 “인 경우도 마찬가지다 하지만 0이나 “은 객체로 평가될 때도 있다.

null 병합 연산자

  1. ES11 에서 도입된 null병합 연산자 ??는 좌항의 피연산자가 null 또는 undefined인 경우 우항의 피연산자를 반환하고 , 그렇지 않으면 좌항의 피연산자를 반환한다.
  2. Null 병합 연산자 ??는 변수에 기본값을 설정할 때 유용하다 .
  3. null 병합 연산자가 도입되기 전에는 논리 연산자 ||를 사용한 단축 평가를 통해 변수에 기본 값을 설정했다.
  4. 논리 연산자 ||를 사용한 단축 평가의 경우 좌항의 피연산자가 false로 평가되는 falsy값 (false, undefined, null, 0 ,-0, NaN, “)이면 우항의 피연산자를 반환한다, 만약 falsy 값인 0 이나 “도 기본값으로서 유효하다면 예기치 않은 동작이 발생할 수 있다.
  5. 하지만 null 병합 연산자 ??는 좌항의 피연산자가 false로 평가되는 Falsy 값(false, undefined, null, 0, -0, NaN, ‘‘)이라도 null 또는 undefined가 아니면 좌항의 피연산자를 그대로 반환한다.
